3161. Container adapters mandate use of emplace_back but don't require it

Section: 21.6.6 [stack], 21.6.4 [queue] Status: New Submitter: Your name Opened: 2018-10-02 Last modified: 2018-10-02

Priority: Not Prioritized

View all other issues in [stack].

View all issues with New status.

Discussion:

21.6.6 [stack] p1 says:
Any sequence container supporting operations back(), push_back() and pop_back() can be used to instantiate stack.

but then in 21.6.6.1 [stack.defn] we have the following code:

template<class... Args>
  decltype(auto) emplace(Args&&... args)
    { return c.emplace_back(std::forward<Args>(args)...); }

The same pattern appears in 21.6.4 [queue].

I believe the calls to c.emplace_back(std::forward<Args>(args)...) should be replaced with c.emplace(c.end(), std::forward<Args>(args)...).

We can do this because emplace is part of the sequence container requirements, while emplace_back is not.

Proposed resolution:

This wording is relative to N4762.

  • Edit 21.6.6.1 [stack.defn], class template stack definition, as indicated:

    template<class... Args>
  decltype(auto) emplace(Args&&... args)
    { return c.emplace_back(c.end(), std::forward<Args>(args)...); }

  • Edit 21.6.4.1 [queue.defn], class template queue definition, as indicated:

    template<class... Args>
  decltype(auto) emplace(Args&&... args)
    { return c.emplace_back(c.end(), std::forward<Args>(args)...); }
